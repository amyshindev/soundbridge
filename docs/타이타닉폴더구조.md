# Titanic 앱 폴더 구조

`backend/apps/titanic/` — Hexagonal + DDD 레이아웃.  
`tests/`, `_docs/`는 이 문서에서 제외.

> **모듈 import:** `from titanic.adapter...` (`PYTHONPATH=apps`)

---

## 1. 전체 트리

```text
titanic/
├── __init__.py
│
├── adapter/                          # 외부 I/O (HTTP, ORM, PG)
│   ├── inbound/
│   │   ├── api/
│   │   │   ├── router_registry.py    # titanic_router — main.py에 등록
│   │   │   ├── schemas/              # Pydantic 요청·응답 스키마
│   │   │   └── v1/                   # FastAPI 라우터 (/titanic/{character}/...)
│   │   ├── grpc/v1/                  # (예약)
│   │   └── websocket/v1/             # (예약)
│   └── outbound/
│       ├── mappers/                  # Entity ↔ ORM/DTO 변환
│       ├── orm/                      # SQLAlchemy 모델, ML 전략
│       └── pg/                       # PostgreSQL repository 구현
│
├── app/                              # 애플리케이션 계층
│   ├── constants/                    # intent_map 등 상수
│   ├── dtos/                         # 유스케이스 입·출력 DTO
│   ├── ports/
│   │   ├── input/                    # Use case 인터페이스 (ABC)
│   │   └── output/                   # Repository port (ABC)
│   ├── use_cases/                    # Interactor (오케스트레이션)
│   ├── db_init.py
│   ├── Titanic-Dataset.csv           # 학습용 CSV (WalterReader 경로)
│   ├── titanic_decision_tree.joblib
│   ├── titanic_decision_tree.pkl
│   └── (레거시) jack_service.py, rose_model.py, james_controller.py,
│                 walter_reader.py, caledon_validation.py
│
├── dependencies/                     # FastAPI Depends DI wiring
│
└── domain/                           # 순수 도메인 (프레임워크·DB 무의존)
    ├── entities/                     # 캐릭터·승객 엔티티
    └── value_objects/                # 피처 VO (Title, Age, Gender, …)
```

## 2. 캐릭터 모듈 명명

| 접두사 | 역할 | stem 예시 |
|--------|------|-----------|
| `crew_*` | 승무원·오피서 | `crew_smith_captain` |
| `passenger_*` | 승객·ML 역할 | `passenger_jack_trainer` |

캐릭터 하나당 아래 파일 stem이 **동일**하게 반복된다.

| 레이어 | 경로 패턴 |
|--------|-----------|
| Router | `adapter/inbound/api/v1/{stem}_router.py` |
| Schema | `adapter/inbound/api/schemas/{stem}_schema.py` |
| DTO | `app/dtos/{stem}_dto.py` |
| Input port | `app/ports/input/{stem}_use_case.py` |
| Output port | `app/ports/output/{stem}_port.py` |
| Interactor | `app/use_cases/{stem}_interactor.py` |
| Provider | `dependencies/{stem}_provider.py` |
| PG repo | `adapter/outbound/pg/{stem}_repository.py` |
| ORM | `adapter/outbound/orm/{stem}_orm.py` |
| Mapper | `adapter/outbound/mappers/{stem}_mapper.py` |
| Entity | `domain/entities/{stem}_entity.py` |

## 4. 레이어별 상세

### 4.1 `adapter/inbound/api/v1/`

```text

passenger_jack_trainer_router.py

passenger_rose_model_router.py
passenger_ruth_validation_router.py
```

→ `router_registry.py`에서 `titanic_router`로 묶어 `main.py`에 mount.

### 4.2 `adapter/inbound/api/schemas/`

캐릭터별 `{stem}_schema.py` + `titanic_passenger_response_schema.py`

### 4.3 `adapter/outbound/pg/`

캐릭터별 `{stem}_repository.py` — `*Port` 구현, AsyncSession 주입.

Walter repository는 DB 대신 `crew_walter_roaster_reader.py`(CSV)를 사용.

### 4.4 `adapter/outbound/orm/`

캐릭터별 `{stem}_orm.py`  
특수: `passenger_rose_model_strategies.py` — sklearn 10모델 전략 + `build_all_strategies()`

### 4.5 `adapter/outbound/mappers/`

캐릭터별 `{stem}_mapper.py` + `titanic_booking_mapper.py`

### 4.6 `app/use_cases/`

## 5. 요청 흐름 (요약)

```text
HTTP  adapter/inbound/api/v1/*_router.py
  →   dependencies/*_provider.py
  →   app/use_cases/*_interactor.py
  →   app/ports/output/*_port.py
  →   adapter/outbound/pg/*_repository.py  (또는 CSV reader / ML strategy)
```

Smith chat 예:

```text
POST /titanic/smith/chat
  → crew_smith_captain_router
  → get_smith_captain_use_case
  → SmithCaptainInteractor.chat()
       Walter → Jack.train → Cal.test → Andrews.analyze_intent → reply
```
